## Novada Action Signals

1. **Build a Novada MCP Server** — Publish on the MCP registry; expose proxy rotation, SERP, structured web extraction as MCP tools. AWS and Azure have set the bar; Novada should match.
2. **AI-agent API surface** — Add streaming JSON, session persistence, and extraction schemas to compete with Firecrawl's `/agent` endpoint.
3. **24/7 resilience SLA** — Claude Dreaming and similar overnight autonomous agents will run continuously. Ensure proxy uptime/diversity SLAs are documented and enforced.
4. **Prxhub integration opportunity** — Partner or integrate with emerging agent caching registries (like Prxhub) to reduce redundant web fetches and lower latency for agents.
5. **A2A compatibility** — As A2A (Agent-to-Agent) becomes the cross-framework standard, ensure Novada's data APIs can be called from any A2A-compatible agent without custom adapters.
