## Top 3 Signals for Novada

### 1. AWS MCP Server Reaches General Availability (May 9)
AWS launched its managed MCP Server into GA, available in US East (N. Virginia) and EU (Frankfurt). It gives AI coding agents secure, auditable access to AWS services via the Model Context Protocol. The ecosystem now counts **10,000+ public MCP servers** under Linux Foundation's Agentic AI Foundation governance, with Streamable HTTP transport, MCP Tasks (SEP-1686), Triggers, and OAuth 2.1 baked into the spec.  
**Novada signal:** Cloud-hyperscaler validation accelerates enterprise MCP adoption. Novada's MCP server offering needs clear differentiation (web-data specialisation, proxy-layer integration) as the baseline rises.

### 2. TikTok Launches Ads MCP Server (May 13)
TikTok opened its ads platform to AI agents through a new MCP server + developer toolkit, enabling automated campaign creation, management, and optimisation.  
**Novada signal:** SaaS giants are self-serving MCP endpoints. Novada should position its proxy/data layer as the missing runtime that spans *across* these first-party servers — handling rate limits, identity rotation, and data normalisation that no single vendor's MCP covers.

### 3. Web Scraping Market on Track for $2B+ in 2026
Firecrawl, Bright Data, Browse AI, and Octoparse dominate AI-powered extraction coverage. Firecrawl's `/agent` endpoint for autonomous research and `/interact` for interactive browser sessions are direct competitive surface. Bright Data leads ethically-sourced proxy scale. Browse AI claims 770,000 users.  
**Novada signal:** Self-healing, context-aware extraction (not structure-dependent) is the new table-stakes feature. Novada's proxy infrastructure is a key differentiator — pitch it as the resilience layer under any scraper.

## Novada Strategic Signals

1. **MCP is the new API layer** — 10,000+ servers, hyperscaler backing, Linux Foundation governance. Novada must have a clear MCP story and ideally a flagship MCP server for web data / proxy access.
2. **Self-healing scrapers reduce proxy commodity pressure** — tools that adapt to site changes reduce the raw proxy volume need. Novada should bundle proxy resilience *with* intelligent retry/rotation as a managed feature.
3. **Local-first trend** (Ollama, llama.cpp, Hermes) — cost-sensitive users will push for on-prem/local agents; Novada's data API should offer lightweight SDK options alongside cloud endpoints.
4. **Firecrawl `/agent` endpoint** is the closest direct competitor to a Novada-style agentic web data API — monitor pricing, rate limits, and data quality gaps.
5. **Enterprise MCP infra question on HN** confirms the buyer conversation is happening now — sales and marketing should engage this thread as a demand signal.
