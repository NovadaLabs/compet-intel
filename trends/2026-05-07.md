## Top 3 Signals for Novada

### 1. AWS MCP Server Goes GA — Cloud Giants Now First-Class MCP Citizens
On May 6, 2026, AWS announced the **general availability of the AWS MCP Server**, giving AI coding agents secure, auditable access to any AWS API through a single MCP tool call. The server supports file uploads, long-running execution, and sandboxed Python scripting against live AWS services. This follows Microsoft bundling MCP natively into Visual Studio 2026 and Agent 365 — meaning the three major cloud platforms now treat MCP as a first-class production protocol, not an experiment.

**Novada angle:** Enterprise agents calling external APIs via MCP still need clean, non-flagged egress IPs when those calls reach public web endpoints. Position Novada's proxy layer as the network substrate that MCP-enabled cloud agents depend on — especially for agents that mix AWS service calls with live web data retrieval in the same workflow.

### 2. CrewAI v1.12 Severs LangChain — Ecosystem Fragments Across 6 New LLM Providers
CrewAI's latest release marks a **complete removal of LangChain** as a dependency and adds native support for six new LLM providers: OpenRouter, DeepSeek, Ollama, vLLM, Cerebras, and Dashscope. The update also ships agent skills, Qdrant Edge memory backend, and hierarchical memory isolation. Meanwhile, LangGraph released v1.1.3 with distributed runtime and deep agent templates. The agent framework layer is now a genuine multi-horse race with no single dependency chain — each runner has different data-access patterns and token budgets.

**Novada angle:** As agents diversify across LLM providers with different performance and cost profiles, demand for high-volume, low-latency web data becomes provider-agnostic. Novada's API should be positioned as a universal data layer — framework-neutral, compatible with any orchestrator that can call an HTTP endpoint or consume an MCP server.

### 3. WebMCP Browser Standard + Firecrawl /interact — Web Interaction Becoming a Native Agent Primitive
Chrome's WebMCP integration (flagged on HN) proposes a browser standard that exposes structured tools for AI agents on existing websites — replacing fragile DOM scraping with a robust protocol layer. Simultaneously, Firecrawl's **/interact** endpoint (GA in 2026) gives agents persistent interactive browser sessions via a single API, and ScrapeGraphAI shipped a **Release Week** (HN featured) with new graph-based AI extraction. The message from the ecosystem: agents interacting with the live web is a solved-enough problem to standardise.

**Novada angle:** As browser-native web interaction matures, the bottleneck shifts from parsing to access — anti-bot measures, geo-restrictions, and rate limits become the primary friction. Novada's residential proxy network is the reliability layer that makes /interact and WebMCP work at scale in production.
