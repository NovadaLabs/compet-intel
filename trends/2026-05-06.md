## Top 3 Signals for Novada

### 1. AI Crawlers Now Dominate Web Traffic — Operators Blocking Entire Countries
Developer discourse on HN this week surfaced a critical inflection point: AI agent crawlers have become the **dominant traffic source** on many public sites, prompting operators to implement geo-blocks and blanket bot-blocking rules that catch legitimate requests. Firecrawl's `/agent` endpoint, Hyperbrowser's new MCP server (launched this week: "Connect AI agents to the web through browsers"), and OpenClaw's 210k-star breakout all rely on unblocked, geo-distributed access to the web.

**Novada angle:** Residential and rotating proxy infrastructure becomes the critical unblocking layer as anti-bot measures intensify. Position Novada as the reliability layer that ensures agent crawls succeed — not as a scraper, but as the clean network path that scrapers depend on.

### 2. MCP Security Layer Emerging — MCP Defender Signals Maturation
The MCP ecosystem crossed 10,000 public servers and this week a "Show HN" for **MCP Defender** (OSS AI Firewall for MCP in Cursor/Claude) generated significant discussion. It proxies all MCP traffic and scans requests and responses in real time — the first dedicated security tool for the MCP layer. This signals the ecosystem is mature enough to warrant threat modelling. Separately, Versium REACH, GoodData, and Easyship MCP servers went GA in Q1, with Azure MCP bundled directly in Visual Studio 2026.

**Novada angle:** Security-conscious teams will prefer MCP servers with observable, trusted data pipelines. Novada's web-data MCP server should emphasise audit logging and request provenance — differentiated from anonymous scraping APIs.

### 3. Claude Opus 4.7 Default + Dispatch Feature → Persistent Agent Sessions Need Fresh Data
Anthropic made **Claude Opus 4.7** the default model on Max and Team Premium tiers, adding an `xhigh` effort level and a live `/effort` slider for coding work. More critically, **Claude Dispatch** lets a single conversation span phone and desktop continuously — users assign tasks on mobile and pick up finished work on desktop, with Claude running scheduled pulls (emails, metrics) in the background. This shifts agent use from one-shot queries to **persistent, scheduled data access**.

**Novada angle:** Scheduled/recurring web data pulls are a new pricing vector. Build a subscription or cron-style access tier for Novada's API to match how Dispatch-style agents actually consume data — not bursts, but steady background intervals.
