## Top 3 Signals for Novada

### 1. Cloud Giants Standardise on MCP — AWS GA + Azure Built-In to VS 2026
AWS announced the **General Availability of the AWS MCP Server** on May 6, 2026, giving AI coding agents secure, auditable access to every AWS API through a single MCP tool — including operations requiring file uploads or long-running execution, and sandboxed Python execution against AWS services. One day earlier, Microsoft confirmed the **Azure MCP Server is now built-in and generally available in Visual Studio 2026**, enabling agentic cloud automation directly from the IDE without manual configuration. Together these announcements mean the two largest cloud platforms now ship MCP natively: MCP has become the default enterprise agent protocol, not an optional plugin.

**Novada angle:** When the two dominant cloud IDEs both ship MCP natively, every enterprise agent developer will expect MCP-compatible data tooling as a first-class integration. Novada's MCP server must be listed in the MCP registry, installable with one command inside Claude Code and VS 2026, and work out-of-the-box with the AWS + Azure MCP stacks. Missing from this ecosystem means missing the enterprise sales motion entirely.

### 2. MCP Proliferates into Vertical Industries — Fintech, Analytics, and Ops Follow Dev Tools
Three significant vertical-industry MCP server launches clustered in early May 2026: **Blend Autopilot MCP Server** (May 4) opens the lending origination platform to financial-institution AI agents; **GoodData MCP Server** (Q1) enables agents to build, update, and operate BI analytics autonomously (10–50× faster than manual BI workflows); **Splunk MCP Server** (early 2026) lets agents query Splunk security data without custom connectors. MCP adoption has crossed the chasm from developer tooling into regulated, data-sensitive verticals: fintech, security ops, and analytics.

**Novada angle:** Agents operating in fintech and security-ops environments need compliant, auditable data access layers that respect IP reputation, geographic restrictions, and rate limits. Novada's proxy API becomes critical infrastructure — not just a developer convenience — in these workflows. Develop compliance messaging (SOC 2, geo-targeting, IP allowlisting) to serve the fintech/ops MCP buyer.

### 3. AI Web Scraping Market Crosses $2B Threshold — Firecrawl, Thunderbit, Browse AI Lead
The global AI web scraping market is now projected to exceed **$2 billion** in 2026. Leading tools driving this: **Firecrawl** (developer/AI-team focused; /agent endpoint for autonomous research; /interact for persistent browser sessions; official MCP server GA); **Thunderbit** (Chrome-first, GTM/sales/e-commerce workflows; 2-click scraping of pages, PDFs, and images); **Browse AI** (770,000+ users; prebuilt and customisable robots). The defining competitive feature in 2026 is "self-healing" extraction — context-aware scrapers that adapt automatically when a target site changes layout. All leading tools depend on reliable proxy layers to get through anti-bot systems.

**Novada angle:** Firecrawl, Thunderbit, and Browse AI are distribution channels for Novada's proxy and web data API. Partnership integrations, co-marketing, or native SDK support inside these platforms would put Novada in front of 770,000+ active web-data users. Specifically: Firecrawl's new /agent and /interact endpoints generate persistent, high-volume browser sessions — exactly the pattern where residential proxy rotation and session management provide the most value.
