## Top 3 Signals for Novada

### 1. Azure MCP Now IDE-Native — Visual Studio 2026 Ships It Built-In
Microsoft confirmed that **Azure MCP Server tools are generally available inside Visual Studio 2026** with no extra install. Every developer on the VS 2026 codebase can now invoke MCP tools without configuring anything. Combined with the **10,000+ public MCP servers** now under Linux Foundation's Agentic AI Foundation governance, MCP has crossed the threshold from emerging standard to infrastructure.

**Novada relevance:** IDE-native MCP means developers will reach for MCP tools before they think of REST calls. Novada's proxy and web-data APIs need to be exposed as first-class MCP tools now — before competitor surfaces occupy the space in agent toolboxes.

### 2. "Code w/ Claude" Conference: Managed Agents, Webhooks & Higher API Limits
Anthropic hosted its annual **Code w/ Claude developer conference in San Francisco this week**, unveiling higher Claude Code and Opus API limits and formalising four **Claude Managed Agents** capabilities: **Dreaming** (cross-session memory), **Outcomes** (goal-rubric execution), **Multiagent Orchestration** (lead agent → specialist delegation), and **Webhooks** (event-driven agent triggers). Claude for Small Business also shipped with 15 prebuilt workflows and integrations for QuickBooks, HubSpot, Canva, and Google Workspace.

**Novada relevance:** Webhook-triggered and memory-aware agents operate autonomously around the clock. They need persistent, schema-stable data pipelines — any instability in the data layer cascades into agent failures. Novada's API reliability story directly addresses this dependency.

### 3. ArXiv: REST APIs Can Be Made Agent-Ready via MCP (2605.14312)
A new paper — **"Making OpenAPI Documentation Agent-Ready"** (May 2026) — shows how AI agents and MCP can automatically expose existing REST APIs as agent-consumable tools, with detection of documentation and REST design smells that block agent adoption. Companion work on **"Agentic Tool Use in LLMs"** (2604.00835) organises the field: prompting-as-plug-and-play → supervised tool learning → reward-driven tool policy.

**Novada relevance:** The paper's methodology is a direct playbook: publish an OpenAPI spec, wrap it in MCP, fix the documentation smells flagged by the tool, and your API becomes discoverable by every major agent framework overnight. Novada should prioritise this now.
