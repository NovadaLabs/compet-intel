## AI Insights

### 1. MCP is the new API distribution war
Every major scraping and search vendor now has an official MCP server except Oxylabs and Decodo. The Model Context Protocol is becoming the primary on-ramp for AI agents — whoever dominates MCP mindshare will own the default data layer for agentic workloads. Novada should audit its own MCP surface and ensure it is listed in Claude's Docker MCP Catalog and in Cursor's plugin ecosystem before the positions calcify.

### 2. Latency is the new moat in agentic search
Exa Fast at <350ms P50 and Firecrawl's tiered Spark models both signal that **sub-second search-to-result** is becoming a hard requirement as LLM agents embed search in tight reasoning loops. Tools that can't respond in under 500ms risk being architectural bottlenecks. Any Novada product with a search or retrieval component should benchmark against this bar.

### 3. Supply chain risk is the sleeper threat for OSS-dependent stacks
The Crawl4AI / litellm supply chain incident (TeamPCP, March 2026) shows that widely-used OSS AI packages are high-value attack targets. Any Novada pipeline that pulls from PyPI should be pinning versions, validating hashes, and monitoring advisories. The 40-minute exposure window before PyPI quarantine is dangerously short — automated dependency scanning is now a baseline requirement.
