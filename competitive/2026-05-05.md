## AI Insights

1. **The Firecrawl Skill is a distribution layer shift, not just a feature.** By allowing Claude Code, Codex, and OpenCode agents to invoke Firecrawl via a `skill://` URI without configuring an MCP server, Firecrawl has created a zero-friction onramp that bypasses the biggest barrier to MCP adoption. Vendors who ship native Skill packages early will accumulate passive distribution through every agent session — Novada should evaluate whether a first-party Skill package belongs on the Q2 roadmap.

2. **SEP-2640 could make the MCP server the universal skill registry.** Apify's PR to serve scrapers as discoverable `skill://` resources through the same MCP connection is a Trojan horse: if ratified, every MCP client gains ambient awareness of what tools are installed. Scraping vendors who register Skills first capture mindshare without requiring developers to explicitly configure them. The spec is still in review, but Novada should prototype a Skills manifest now — the window to shape early adoption norms is narrow.

3. **Token economics are the new pricing battleground in agentic retrieval.** Exa's query-dependent highlights reduce per-query token cost 50–75 % while Spark 1 Fast prices agent retrieval at a flat 10 credits/cell. Both moves compress the per-query economics of agentic pipelines, creating downward pressure on incumbent search and scraping providers who charge per result regardless of relevance. Products that bundle retrieval, relevance filtering, and formatting into a single low-cost call will displace point solutions that force customers to assemble the stack themselves.
