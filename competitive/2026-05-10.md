## AI Insights

1. **Firecrawl's Skill + CLI combo is the most strategically significant distribution move in the space this week.** Publishing a Firecrawl Skill that auto-discovers in Claude Code means every developer opening Claude Code for a web task sees Firecrawl as the suggested tool before they've searched for alternatives. CLI + Skill together cover the two most important non-browser developer touchpoints: terminal workflows and AI coding agents. Novada's KR-3 (Claude Plugin, LangChain/CrewAI PRs) targets the right channels but is behind on the Claude Code side specifically. The window to publish a novada-search Claude Code skill before Firecrawl's installed base becomes entrenched is weeks, not months.

2. **Decodo in LangChain's official JavaScript docs changes the KR-3 landscape materially.** When Novada's LangChain PR lands, it will be adding a second integration in the scraping/proxy category, not claiming the first slot. That's a harder positioning narrative with LangChain maintainers and harder to surface in developer search. The Python docs slot is unconfirmed — that's the one to audit and claim first. Decodo's integration is gated behind "Web API Advanced plans" only, which means a free-tier or open-access Novada integration could still win developer mindshare on the same docs page.

3. **Firecrawl's Spark models create proprietary lock-in that Novada's model-agnostic stack doesn't have — publish this as a differentiator before it's assumed to be a gap.** Spark 1 Fast/Mini/Pro are closed models optimized for Firecrawl's own agent endpoint. Developers building on Firecrawl's `/agent` are now dependent on Firecrawl's model roadmap, pricing, and availability. novada-search's architecture (search + proxy, no proprietary model layer) means developers keep full model choice. This is a genuine architectural advantage for teams that want model portability. It's not being communicated — and absent a published statement, developers will assume Novada just hasn't built models yet.
