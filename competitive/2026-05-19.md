## AI Insights

### 1. FIRE-1 in MCP closes the last gap between browsing agents and paywalled data
Firecrawl's May 17 MCP update — adding FIRE-1 model access to `/scrape` and `/extract` — means Claude Code and Cursor agents can now reach behind logins, button triggers, and modals without custom scraping code. This is a qualitative capability jump: previously MCP scraping was limited to publicly accessible HTML. Competitors like BrightData expose browser automation via MCP but require explicit session management. Novada should evaluate whether offering authenticated-page extraction as a first-class product feature (not just a power-user capability) becomes a table-stakes requirement within 2–3 quarters.

### 2. The OSS scraping stack has a systemic supply-chain problem — and it's accelerating
Crawl4AI's v0.8.6 hotfix (PyPI litellm backdoor, SSH/cloud credential harvest) is the second major OSS scraping security event in 2026. CVE-2026-26216 (Docker RCE, CVSS 10.0) remains open for legacy deployments, and v0.8.0 itself shipped with two critical Docker vulns. The pattern: fast-moving OSS scraping libraries accumulate dependencies, dependency trees become attack surfaces, and security response lags feature velocity. For enterprise buyers evaluating Crawl4AI vs managed scraping APIs, this is now a documented risk — not a theoretical one. Novada can position managed, audited infrastructure as the credible alternative, especially for customers in regulated industries.

### 3. The proxy IP arms race is plateauing — differentiation is shifting to intelligence quality
Oxylabs grew its residential pool from 72M to 100M IPs in Q1. BrightData claims ~150M. Decodo holds 125M+. The marginal value of additional raw IPs is compressing as all major providers now exceed enterprise-scale coverage. The next differentiator is intelligence quality: AI-powered unblocking (Oxylabs), AI Parser / structured output (Decodo), Exa's neural index freshness. Raw proxy capacity is becoming a commodity input; the winners will be those who layer semantic understanding, structured extraction, or domain-specific signals on top. Novada's product roadmap should reflect this shift — competing on IP count is a losing game; competing on data quality and entity resolution is where pricing power lives.
