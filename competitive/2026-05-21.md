## AI Insights

### 1. Exa's $250M signals AI-native search as critical infrastructure
The raise — announced just yesterday — follows a clear pattern: neural/semantic search is being valued as foundational infrastructure, not a feature. At $2.2B, Exa is pricing in the assumption that every AI agent will need high-quality, real-time web retrieval. This directly threatens commodity scraping plays and raises the bar for what "good enough" retrieval means. Novada should evaluate whether its retrieval quality and freshness can compete at the level Exa 2.0 promises (sub-350ms, minute-level index refresh, field-level structured outputs).

### 2. MCP token efficiency is the new battlefield
BrightData's reduction from 60+ MCP tools to 2 — after measuring that 90% of agent calls used only search + scrape — shows that MCP integration quality is now measured by token cost, not feature count. Exa consolidated from 9 to 4 tools for the same reason. Any competitor shipping a bloated MCP server will lose to leaner alternatives as enterprise customers optimize agent costs. Novada's MCP offering (if any) should audit tool count, schema verbosity, and per-call token overhead.

### 3. Open-source is commoditizing basic crawling; value is moving up the stack
Crawl4AI at 58K stars, with anti-bot proxy escalation, Shadow DOM flattening, crash recovery, and MCP support, is effectively free for any developer comfortable with self-hosting. This squeezes pure-play commercial crawlers. The commercial winners — Firecrawl's Lockdown Mode, Exa Deep's agentic retrieval, BrightData's ARIA browser automation, Apify's LLM extract endpoints — are all adding AI-native value layers on top of crawling primitives. Novada's product differentiation must live above the crawl layer.
