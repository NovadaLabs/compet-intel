## Per-Competitor Updates

### 1. Firecrawl
- **`/extract` officially deprecated**: JS and Python SDK `extract()` methods marked deprecated; `/agent` is now the canonical extraction and deep-research endpoint. The successor `/agent` is described as "faster, more reliable, and doesn't require URLs — just describe what you need."
- **Firecrawl CLI launched**: Full command support — `scrape`, `search`, `crawl`, `map`. Developers can run Firecrawl from a terminal without any SDK.
- **Skill package (skill://)**: Claude Code, Codex, and OpenCode can invoke Firecrawl autonomously using `skill://` — no MCP config required; the skill is self-contained and discoverable by agent runtimes.
- **GET /v2/team/activity endpoint**: Lists recent scrape/crawl/extract jobs with cursor-based pagination (last 24 h, up to 100 results/page, filterable by endpoint type). Useful for monitoring agent job history.
- **Spark 1 Fast now in Playground**: Previously unreleased publicly; Spark 1 Fast (instant retrieval, 10 cr/cell) now testable in the Playground UI.
- **Parallel Agents**: thousands of simultaneous `/agent` calls with automatic waterfall failover (Fast → Mini).
- **MCP v3 stable**: Cloud MCP with HTTP Transport and SSE modes; `firecrawl_agent` + `firecrawl_agent_status` (from May 5 update) now used in production.

## AI Insights

1. **Firecrawl's `/extract` deprecation resets the extraction API conversation.** Moving from `/extract` (URL-in, schema-out) to `/agent` (intent-in, results-out) is a paradigm shift, not just a rename. Developers building new integrations today will learn the `/agent` mental model first — agents describe what they want, the AI figures out where to look. Any product that ships an `/extract`-style endpoint now is presenting 2025 UX to a 2026 developer audience. The urgency for Novada is to ensure its Extract endpoint (pending infra activation) either matches this intent-driven model or is framed explicitly as a complement to it.

2. **BrightData's Claude ecosystem content blitz is a distribution strategy, not just marketing.** Publishing "Claude Agent SDK + Web MCP", "Claude Code + Web MCP", and "Claude Skills + Web MCP" guides in quick succession is a systematic play to own the SEO surface area for "[Claude tool] + web data" — the exact queries Novada's target customers will run when evaluating options. BrightData's MCP is already published and functional; every guide they publish while novada-proxy is pending approval is a compounding distribution advantage. The window to publish first-mover content in this space is closing.

3. **Tavily going upmarket under Nebius is a structural opening for developer-first search.** IBM THINK attendance and the April product quiet period (events, AMAs, no feature announcements) signal that Tavily is in enterprise repositioning mode. The developer-first, self-serve, affordable search API segment — Tavily's historic stronghold — is temporarily under-invested. This is the same moment Novada's Search endpoint should be activating. If novada-search can ship with a clear developer-first narrative (transparent pricing, fast integration, MCP-native) while Tavily is enterprise-focused, there is a genuine positioning window to capture Tavily's organic developer community.
