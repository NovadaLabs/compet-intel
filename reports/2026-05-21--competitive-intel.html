Insight 02 洞察 02
MCP token efficiency, not feature count, is the new competitive metric MCP token效率而非功能数量，才是新的竞争衡量标准
BrightData measured that 90% of agent calls used only 2 of their 60+ MCP tools — and the 60-tool handshake cost ~17,000 tokens before any real work. They cut down to 2. Exa reduced from 9 to 4 tools for the same reason. As enterprise customers optimize agentic workflow costs, bloated MCP schemas become a liability. The winning MCP integration is not the most capable — it's the most efficient. Any Novada MCP offering should be audited for tool count, schema verbosity, and per-invocation overhead before shipping. BrightData的测量发现，90%的智能体调用只使用了60+个MCP工具中的2个，而60个工具的握手过程在真正开始工作前就消耗了约17,000个token。他们因此精简至2个工具。Exa出于同样原因从9个工具减少到4个。随着企业客户优化智能体工作流成本，臃肿的MCP schema正成为负担。最终胜出的MCP集成不是功能最全的，而是效率最高的。Novada的任何MCP产品在发布前都应对工具数量、schema冗余度和每次调用开销进行审计。
