Claude Opus 4.7 "Dreaming" + Multiagent Orchestration
Claude Opus 4.7「梦想」+ 多代理编排
Anthropic shipped four capabilities that define the next agent generation: Dreaming (scheduled agents that review past sessions and self-improve between runs), Multiagent Orchestration (lead agent delegates tasks to parallel sub-agents on a shared filesystem), Outcomes (auto-grading loop that sends agents back to revise until a rubric passes), and a new Claude Code agent view for managing concurrent sessions. Always-on, self-improving agents need reliable, low-latency web data APIs. Novada must validate MCP compatibility with Claude Managed Agents and test Dreaming/Routines invocations now.
Anthropic 推出四项定义下一代代理的能力：梦想（计划代理自动回顾过往会话并在运行间自我优化）、多代理编排（主代理在共享文件系统上将任务分配给并行子代理）、成果（自动评分循环，直到满足评分标准才停止修订）以及全新 Claude Code 代理视图（管理并发会话）。始终在线、自我改进的代理需要可靠的低延迟网络数据 API。Novada 必须立即验证与 Claude 托管代理的 MCP 兼容性，并测试梦想/例程调用。

"Trust No Tool" + MCP RCE = Security is a Sales Argument
「不信任工具」+ MCP RCE = 安全成为销售论点
A May 17 arXiv paper (2605.17453) proves LLM agents blindly trust tool feedback, making tampered responses a viable supply-chain attack vector. This is compounded by an April 2026 MCP design flaw enabling RCE across 7,000+ servers and 150M downloads. Enterprises choosing a web-data MCP provider will now ask: "Can I trust the data coming back from this tool?" Novada has an immediate opportunity to differentiate with signed, timestamped, provenance-tagged API responses — and a published security posture document.
5 月 17 日 arXiv 论文（2605.17453）证明 LLM 代理盲目信任工具反馈，使篡改响应成为可行的供应链攻击向量。2026 年 4 月的 MCP 设计漏洞更进一步，在 7,000+ 服务器和 1.5 亿次下载中启用远程代码执行。选择网络数据 MCP 提供商的企业现在会问：「我能信任从该工具返回的数据吗？」Novada 有立即差异化的机会——带签名、时间戳和来源标记的 API 响应，以及公开的安全立场文档。

主流 SaaS 平台代理化：Notion + TikTok（5月13日）

Notion 推出带 Workers 的外部代理 API（Notion 内部自定义代码 + 多步骤自动化工作流）。TikTok 推出广告 MCP 服务器，用于自主广告创建和优化——两者同日发布。这一模式正在加速：每个主要 SaaS 都在暴露代理集成接口。这些平台代理需要外部网络情报才能行动。Novada 的代理和爬取 API 应被定位为支撑代理化 SaaS 工作流的网络数据基础层。

Security Risks in Tool-Enabled AI Agents (IEEE COMPSAC 2026) — 2605.09721 (May 10)
Cloud-hosted agents in privileged execution environments performing side-effecting operations via tools face novel attack surfaces. Validates demand for auditable, access-controlled data APIs — not just performant ones.
在特权执行环境中通过工具执行副作用操作的云托管代理面临新型攻击面。验证了对可审计、访问控制数据 API 的需求 — 不仅仅是高性能的。

Agent-X: Full Pipeline Acceleration of On-device AI Agents — 2605.10380 (May 11)
Software-only framework accelerating both prefill and decode for on-device agent workloads. On-device and local-first agent usage is growing — users in this segment need proxy APIs that work without cloud dependencies.
加速设备端代理工作负载预填充和解码阶段的纯软件框架。设备端和本地优先代理使用正在增长 — 该细分市场用户需要无云依赖的代理 API。
