AWS MCP Server Goes GA — Cloud Giants Now First-Class MCP Citizens

AWS MCP Server 正式发布——云巨头成为 MCP 一级公民

On May 6, AWS announced GA of the AWS MCP Server, giving AI coding agents secure, auditable access to any AWS API via a single MCP tool call — including sandboxed Python execution. Microsoft bundled MCP natively in Visual Studio 2026 and Agent 365. All three major cloud platforms now treat MCP as a production protocol.

5月6日，AWS 宣布 AWS MCP Server 正式发布，使 AI 编码代理可通过单一 MCP 工具调用安全、可审计地访问所有 AWS API，并支持沙盒 Python 执行。微软已将 MCP 原生集成到 Visual Studio 2026 和 Agent 365 中，三大云平台均将 MCP 定为生产级协议。

Novada Angle
Enterprise agents calling web data inside AWS workflows need clean egress IPs. Position Novada's proxy layer as the trusted network substrate beneath MCP-enabled cloud agents. 在 AWS 工作流中调用外部网络数据的企业代理需要干净的出口 IP。将 Novada 的代理层定位为 MCP 云代理的可信网络基础设施。

CrewAI v1.12 Severs LangChain — Ecosystem Fragments Across 6 New LLM Providers

CrewAI v1.12 彻底移除 LangChain——生态系统跨 6 个新 LLM 提供商分裂

CrewAI v1.12 fully removes LangChain and adds native support for OpenRouter, DeepSeek, Ollama, vLLM, Cerebras, and Dashscope, plus agent skills and Qdrant Edge memory. LangGraph v1.1.3 adds distributed runtime. The agent framework layer is now a genuine multi-horse race.

CrewAI v1.12 完全移除 LangChain，新增对 OpenRouter、DeepSeek、Ollama、vLLM、Cerebras 和 Dashscope 的原生支持，以及智能体技能和 Qdrant Edge 记忆后端。LangGraph v1.1.3 增加分布式运行时。智能体框架层现已成为真正的多方竞争格局。

Novada Angle
Framework-agnostic positioning is critical. Novada's API must be the universal data layer across all orchestrators — compatible with any framework that can call an HTTP endpoint or consume an MCP server. 框架中立定位至关重要。Novada 的 API 必须成为跨所有编排框架的通用数据层——兼容任何可调用 HTTP 端点或使用 MCP 服务器的框架。
Signal 03
信号 03

WebMCP Browser Standard + Firecrawl /interact — Web Interaction Becomes a Native Agent Primitive

WebMCP 浏览器标准 + Firecrawl /interact——网页交互成为代理原生能力

Chrome's WebMCP integration proposes a browser-native standard exposing structured tools for AI agents, replacing fragile DOM scraping. Firecrawl's /interact GA and ScrapeGraphAI's HN Release Week signal that live web interaction is mature enough to standardise. The ecosystem is solving the parsing problem; access is the new bottleneck.

Chrome 的 WebMCP 集成提出了一个浏览器原生标准，为 AI 代理暴露结构化工具，取代脆弱的 DOM 抓取。Firecrawl /interact 正式发布和 ScrapeGraphAI 的 HN 发布周表明，实时网页交互已足够成熟可以标准化。生态系统正在解决解析问题，访问才是新的瓶颈。

Novada Angle
As browser-native interaction matures, anti-bot measures, geo-restrictions, and rate limits become the primary friction. Novada's residential proxy network is the reliability layer that makes /interact and WebMCP work at production scale. 随着浏览器原生交互成熟，反机器人措施、地理限制和速率限制成为主要阻力。Novada 的住宅代理网络是使 /interact 和 WebMCP 在生产规模下正常运行的可靠层。
