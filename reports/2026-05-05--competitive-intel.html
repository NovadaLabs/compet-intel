AI Insights AI 洞察

01

Firecrawl Skill Is a Distribution Layer Shift

By letting Claude Code, Codex, and OpenCode invoke Firecrawl via a skill:// URI without any MCP server setup, Firecrawl has removed the biggest barrier to agent-native adoption. Vendors who ship native Skill packages early will accumulate passive distribution through every agent session — Novada should evaluate a first-party Skill package for the Q2 roadmap before this channel matures.

Firecrawl Skill 是分发层的范式转变

通过让 Claude Code、Codex 和 OpenCode 无需配置 MCP 服务器即可调用 Firecrawl，Firecrawl 消除了智能体原生采用的最大障碍。提前推出原生 Skill 包的厂商将在每次智能体会话中获得被动分发——Novada 应在该渠道成熟之前评估将自有 Skill 包纳入 Q2 路线图。

02

SEP-2640 Could Make MCP the Universal Skill Registry

Apify's PR to serve scrapers as discoverable skill:// resources is a structural play: if ratified in the MCP spec, every MCP client gains ambient awareness of what tools are available — no explicit configuration needed. Vendors who register Skills first capture mindshare before the standard hardens. The window to shape early adoption norms is narrow; prototype a Skills manifest now.

SEP-2640 或将 MCP 变为通用技能注册中心

Apify 将爬虫工具作为可发现 skill:// 资源的提案是一步结构性棋局：一旦纳入 MCP 规范，每个 MCP 客户端都将自动感知可用工具——无需显式配置。率先注册 Skills 的厂商将在标准固化前占据心智高地。塑造早期采用规范的窗口期很短，现在就应着手准备 Skills 清单原型。

03

Token Economics Are the New Pricing Battleground

Exa's query-dependent highlights compress per-query token costs 50–75 %, while Firecrawl's Spark 1 Fast prices agent retrieval at a flat 10 credits/cell versus full agentic pricing. Both moves set a new market expectation: retrieval + relevance filtering should be bundled and cheap. Providers that still charge per raw result regardless of relevance face structural margin compression as token-efficiency becomes a primary evaluation criterion for agentic pipeline buyers.

Token 经济学成为新定价战场

Exa 的查询相关摘要将每次查询的 token 成本压缩 50–75%，而 Firecrawl Spark 1 Fast 以每格 10 积分的固定价格提供智能体检索。两者共同树立了新的市场预期：检索 + 相关性过滤应当捆绑且低价提供。仍按原始结果数量计费而不考虑相关性的提供商，随着 token 效率成为智能体管道买家的核心评估标准，将面临结构性利润压缩。
